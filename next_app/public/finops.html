Hallazgo R5 · Unidad de margen
· Literatura existente ·

La literatura de optimización LLM (FrugalGPT, RouteLLM) define restricciones de margen sobre requests individuales. Abu Oracle demuestra empíricamente que esta formulación es infactible para planes de bajo ingreso con rutas de alta complejidad doctrinal: el costo de un único request Sonnet supera el ingreso por request del plan anual.

Ingreso anual/req = $45 ÷ (365 × 10 req/día) = $0.0123/req
Costo Sonnet/domain = $0.012–0.015/req → R5 estructuralmente infactible
· Contribución Abu Oracle ·

La unidad de análisis correcta es la sesión. Una sesión típica combina rutas de alto costo (Sonnet/domain) con rutas de bajo costo (Haiku/technique), produciendo margen agregado positivo aunque requests individuales sean negativos. Esta reformulación — Σ(margeni × freqi) ≥ margen_mínimo_sesión — distingue el modelo de Abu Oracle de los enfoques existentes.
